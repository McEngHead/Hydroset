### Req 3 — 복잡한 합류점 배치 개선

**파일:** `06.하도추적.py`
**위치:** `NetworkCanvas._auto_layout()`, Step 7 앞에 Step 6.5 삽입

**문제:** 서브 합류점(GSC00)의 reach-연결 소유역(GSC04)이 Y=200에 수평으로 배치될 때,
같은 X 위치의 본류 합류점(MG18, MG17)도 위쪽에 소유역을 배치하려 해서 선이 겹침.

**해결 — forced_below_mcs 집합 구성:**

```
Step 6.5: 서브 합류점 reach-소유역이 점유할 X 위치 사전 계산
  - 서브 합류점 자체가 mc_x와 같은 X → 해당 mc_id를 forced_below에 추가
  - reach-소유역이 (sj_x - STEP_X)에 올 X → 그 X를 가진 mc_id도 forced_below에 추가
Step 7: force_below인 mc_id는 소유역 전부 아래로 배치 (교번 없음)
```

**Sample_Redraw.json 기준 예상 결과:**
```
Y=  40:  XGSC00(720)                    ← GSC00의 수직 소유역 (위)
Y= 240:  MG24(320) GSC04(520) GSC00(720) XMG15(920) SYC00(1120) XMG13(1320) SW01(1720)
Y= 440:  MG23─MG18─MG17─MG15─MG14─MG13─RS01─SW00─OUT  ← 본류 수평
Y= 640:  XMG23(320) XMG18(520) XMG17(720) XMG14(1120) XSW00(1720)
```
- MG17(X=720): GSC00이 위를 차지 → XMG17 아래(640) ✓
- MG18(X=520): GSC04가 위를 차지 → XMG18 아래(640) ✓
- 선 겹침 없음

### Req 4 — Ctrl+Z / Ctrl+Alt+Z 복원

**문제:** `NetworkCanvas`에 `<Control-z>` 바인딩은 존재하나, 캔버스가 포커스를 갖지 않으면 동작 안 함.
`<Control-Alt-z>`(Redo) 바인딩이 없었음.

**변경 1 — NetworkCanvas.__init__:**
```python
self.bind('<Control-Alt-z>', self._redo)   # 추가
```

**변경 2 — NetworkEditorWindow.__init__ (창 수준):**
```python
self.bind('<Control-z>',     lambda e: self._canvas._undo())
self.bind('<Control-Alt-z>', lambda e: self._canvas._redo())
```
→ 어느 위젯에 포커스가 있어도 Undo/Redo 동작.

### Req 6 — 하도추적 요소 삽입 방식

**이전 방식:** 팔레트 하도추적 아이콘 클릭 → 포트 클릭 → 포트 클릭 (새 reach 연결선 생성)

**새 방식:**
1. 단순 연결선을 선택
2. 팔레트 하도추적 아이콘 클릭 → 선택된 연결선이 하도추적 연결선으로 즉시 변환
3. 기본값: K=1.0, X=0.20, NSTPS=0, label=`REACH_{edge_id}`
4. 이후 속성창에서 K, X 값 수정 가능

**연결선이 선택되지 않은 경우:** 기존 포트 클릭 연결 모드 유지 (하위 호환)

**팔레트 아이콘 하단:** "연결선 선택 후 클릭" 힌트 레이블 추가

### Req 7 — 불러오기 멀티 포맷

**파일 선택 다이얼로그:** `.json`, `.dat`, 또는 둘 다 다중 선택 가능 (`askopenfilenames`)

**로드 로직:**

| 선택 파일 | 동작 |
|---|---|
| .json 만 | 기존과 동일 (JSON → canvas state) |
| .dat 만 | DAT 파싱 → ops 리스트 → `load_operations()` → `_auto_layout()` → `_zoom_extents()` |
| .json + .dat | JSON으로 canvas 복원 (위치 정보 우선), DAT 내용은 미리보기 패널에 표시 |

**DAT 파서 (`_parse_dat_ops` static method):**
- HEC-1 카드 기반 파싱: `KK`(이름), `BA`(면적), `PB`(강우), `LS`(CN), `UC`(Tc/R)
- `RM`(머스킹엄 → ROUTE op), `HC`(합산 → COMBINE op)
- 기타 카드 무시 (IO, IT, VS, VV, PC 등)
